Manual de uso - Git/GitHub

Este site tem como objetivo ser um How to use de Git e GitHub. Use-o como um guia rápido.

init Cria um repositório Git vazio ou reutiliza um já existente.
clone Clona um repositório em um novo diretório.
branch Lista, cria ou remove branches.
checkout Troca branches ou restaura arquivos da Working Tree.
status Mostra o status da Working Tree.
diff Mostra a diferença entre commits, arquivos, branches e a Working Tree.
add Adiciona os conteúdos do arquivo para o index (staging area).
commit Registra alterações no repositório.
push Envia alterações locais para um repositório remoto.
pull Busca e integra com outro repositório ou uma branch local.
revert Reverte algumas commits existentes.
merge Combina alterações de diferentes branches.
stash Guarda as alterações em um diretório de trabalho.
rm Remove arquivos da Working Tree e do index (staging area).
config Configura opções globais ou do repositório.
reset Retorna a HEAD atual para o estado especificado.
remote Gerencia repositórios remotos.
fetch Obtém branches e/ou tags de um ou mais repositórios.
show Mostra um ou mais objetos (tags, commits, etc).
help Mostra informação de ajuda sobre o Git.
rebase Reaplica commits em cima de outra base.
blame Mostra qual alteração e autor modificou cada linha de código pela última vez.
tag Cria, lista, apaga ou verifica uma tag.

Criando um repositório:

  1. No canto superior direito de qualquer página, use o menu suspenso e selecione "New repository".
  2. No campo "Repository name", digite o <nome_do_repo>.
  3. No campo "Description", digite uma pequena descrição.
  4. Selecione se seu repositório será Público ou Privado.
  5. No campo "Repository name", digite o <nome_do_repo>.
  6. No campo "Repository name", digite o <nome_do_repo>.

Criando uma branch:

  1. Clique na aba "Code" do seu repositório <nome_do_repo>.
  2. Acima da lista de arquivos, clique no menu suspenso que diz "main".
  3. No campo de texto, digite o nome da branch, por exemplo, <readme-edits>.
  4. Clique em "Create branch: readme-edits from main".

    Pronto, agora você tem duas branches idênticas: <readme-edits> e <main>

Fazendo e aplicando alterações:

  1. Na branch <readme-edits> criada anteriormente, clique no arquivo README.md.
  2. Para editar o arquivo, clique no ícone de lápis.
  3. No editor, escreva o conteúdo do arquivo.
  4. Clique em "Commit changes...".
  5. No campo "Propose changes", escreva uma mensagem de commit que descreva suas alterações.
  6. Clique em "Propose changes".
